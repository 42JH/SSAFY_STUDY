# 자료구조 스터디

**선형 자료구조: 스택, 큐, 덱**

- 데이터가 일렬로 나열된 형태. 데이터를 넣고 빼는 순서가 가장 큰 차이점
1. **큐: “공평한 줄 서기”**
- 큐는 가장 먼저 들어온 데이터가 가장 먼저 나가는 FIFO(First-In, First-Out) 구조입니다.

- 파이썬 구현: collections.deque 를 사용합니다. list 의 pop(0) 은 데이터 이동 때문에
  O(N)의 시간이 걸리지만, deque 는 O(1)로 처리가 가능합니다.

- **주요 메서드**
  
  - append(x) : 뒤에 데이터 추가 (입구)
  - popleft() : 앞에서 데이터 제거 및 반환 (출구)
  - 실무 활용: 은행 번호표 시스템, 프린터 출력 대기열, 너비 우선 탐색(BFS) 알고리즘.
1. **덱: “양방향 끝판왕”**

덱은 이름처럼 양쪽 끝(Double-ended)에서 삽입과 삭제가 모두 가능한 자유로운 자료구조입니다.

- 특징: 스택처럼 쓸 수도 있고, 큐처럼 쓸 수도 있는 유연함이 장점입니다. 파이썬의 deque 는 내부적
  으로 양방향 연결 리스트(Doubly Linked List)로 구현되어 있어 양 끝단 작업이 매우 빠릅니다.

- 주요 메서드
  
  - append(x) / appendleft(x) : 오른쪽/왼쪽 삽입
  - pop() / popleft() : 오른쪽/왼쪽 삭제
  - rotate(n) : 요소를 회전시키는 독특한 기능 (양수면 오른쪽, 음수면 왼쪽 이동)

- 실무 활용: 스크롤 히스토리(최근 본 페이지), 팰린드롬(회문) 검사, 슬라이딩 윈도우 알고리즘.
1. 힙: “우선순위의 질서”

힙은 여러 값 중 최댓값이나 최솟값을 빠르게 찾아내기 위해 고안된 완전 이진 트리 기반의 자료구조입니
다.

- 파이썬 구현: heapq 모듈을 사용합니다. 별도의 객체를 만드는 게 아니라, 일반 리스트를 힙처럼 다
  루게 해줍니다.
- 핵심 원리:
- 최소 힙(Min Heap): 부모 노드가 자식 노드보다 항상 작습니다. 따라서 인덱스 0 은 항상 전체
  의 최솟값입니다.
- 파이썬은 기본이 최소 힙입니다. (최대 힙이 필요하면 값에 마이너스 - 를 붙여서 저장하는 트릭
  을 씁니다.)
- 주요 메서드
  - heapq.heappush(list, val) : 힙 성질을 유지하며 값 추가
  - heapq.heappop(list) : 가장 작은 값을 제거하고 반환

**실무 활용**: 우선순위 큐 구현, 다익스트라 최단 경로 알고리즘, 실시간 랭킹 시스템.

**rotate() 는 파이썬 collections.deque 만이 가진 아주 매력적이고 강력한 기능입니다. 리스트에서
는 요소를 한 칸씩 옆으로 밀려면 반복문을 돌리거나 슬라이싱을 복잡하게 해야 하지만, 덱에서는 메서드
하나로 해결됩니다.**

힙, 큐

```python
import heapq

nums = [10, 20, 30]
max_heap = []

for n in nums:
    heapq.heappush(max_heap, -n)  # 마이너스 붙여서 저장

# 뺄 때 다시 마이너스 붙이기
print(-heapq.heappop(max_heap))  # 결과: 30 (최대값)
```

**언제 어떨때 써야될까?**

- 순서가 중요하고 앞 뒤로 자주 넣고 뺀다 — 덱
- 가장 작거나 큰 걸 계속 찾아야 한다? — 힙
- 데이터를 키값으로 바로바로 찾고싶다? — 딕셔너리
- 중복을 없애고 집합 연산을 해야한다?? — 집합(set)
