# [D4] 5204. 병합 정렬

[문제 링크](https://swexpertacademy.com/main/code/problem/problemDetail.do?contestProbId=AWT-Pu3acjYDFAVT)

## 📝 문제 설명

$N$개의 정수를 가진 리스트를 **병합 정렬(Merge Sort)**을 사용하여 정렬할 때, 병합 과정에서 다음과 같은 특정 조건을 만족하는 횟수를 구하는 문제입니다.

- 왼쪽 부분 리스트의 마지막 원소가 오른쪽 부분 리스트의 마지막 원소보다 큰 경우의 개수를 출력합니다.
- 최종적으로 정렬된 리스트의 $N//2$번째 원소와 해당 카운트를 출력해야 합니다.

---

## 🔍 알고리즘: 분할 정복 (Divide & Conquer)

이 문제는 전형적인 **분할 정복** 기법인 병합 정렬을 구현하는 문제입니다. 리스트를 더 이상 나눌 수 없을 때까지 절반으로 나누고, 다시 합치면서 정렬을 수행합니다.

### **분할 정복의 과정**

1. **Divide (분할)**: 해결할 문제를 리스트의 중간 지점을 기준으로 두 개의 하위 문제로 나눕니다.
2. **Conquer (정복)**: 각 하위 문제를 재귀적으로 해결합니다. (크기가 1이 될 때까지)
3. **Combine (병합)**: 정렬된 두 하위 리스트를 하나의 정렬된 리스트로 합칩니다.

---

## ⚙️ 입력 및 출력

### **입력**

- 첫 줄에 테스트 케이스의 개수 $T$가 주어집니다.
- 각 테스트 케이스의 첫 줄에는 $N$ ($10 \le N \le 1,000,000$)이 주어집니다.
- 다음 줄에는 $N$개의 양의 정수가 공백을 사이에 두고 주어집니다.

### **출력**

- `#{test_case} {N//2번째 원소} {카운트}` 형태로 출력합니다.

---

## 💡 풀이 전략

1. **재귀 구조 설계**:
   - `merge_sort` 함수를 만들어 리스트의 길이가 1이 될 때까지 `low`, `mid`, `high`를 이용해 재귀 호출합니다.
2. **병합 로직 (Merge)**:
   - 두 리스트를 합칠 때, 왼쪽 리스트의 마지막 값과 오른쪽 리스트의 마지막 값을 비교하여 문제의 조건을 체크합니다.
   - `if left[-1] > right[-1]: count += 1`
3. **최적화**:
   - 파이썬의 경우 `list.pop(0)`이나 슬라이싱을 과도하게 사용하면 시간 초과가 발생할 수 있습니다.
   - 인덱스를 직접 관리하거나 `collections.deque`를 고려해야 하며, 병합 결과를 담을 새로운 배열을 효율적으로 관리해야 합니다.

---

## ⚠️ 주의 사항

- **메모리 및 시간 제한**: $N$이 최대 1,000,000이므로 단순한 리스트 슬라이싱 방식은 메모리 초과나 시간 초과를 유발할 수 있습니다.
- **인덱스 관리**: $N//2$번째 원소를 찾는 것이므로 0-indexed 배열 기준으로 정확한 위치를 파악해야 합니다.

# [D4] 3074. 입국심사

[문제 링크](https://swexpertacademy.com/main/code/problem/problemDetail.do?contestProbId=AV_XEokaAEcDFAX7)

## 📝 문제 설명

$N$개의 입국 심사대가 있고, 각 심사대마다 심사하는 데 걸리는 시간 $T_i$가 다릅니다. $M$명의 사람이 모든 심사를 마치는 데 걸리는 **최소 시간**을 구하는 프로그램을 작성하시오.

---

## 🔍 알고리즘: 이진 탐색 (Binary Search)

이 문제는 "시간 $X$ 내에 모든 사람을 심사할 수 있는가?"라는 **결정 문제**로 바꾸어 해결하는 **매개 변수 탐색(Parametric Search)** 유형입니다.

### **이진 탐색의 핵심 원리**

1. **탐색 범위 설정**:
   - **Low**: 1 (최소 시간)
   - **High**: (가장 오래 걸리는 심사대 시간 $\times M$)
2. **중간값(Mid) 검사**: 특정 시간 `mid`가 주어졌을 때, 각 심사대가 처리할 수 있는 사람 수의 합을 구합니다.
   - 각 심사대가 `mid` 시간 동안 처리 가능한 인원: `mid // T_i`
3. **범위 조절**:
   - 총합이 $M$명 이상이면: 시간이 남는 것이므로 더 최소 시간을 찾기 위해 `high = mid - 1`
   - 총합이 $M$명 미만이면: 시간이 더 필요하므로 `low = mid + 1`

---

## ⚙️ 입력 및 출력

### **입력**

- 테스트 케이스 개수 $T$
- $N$ (심사대 수, $1 \le N \le 100,000$), $M$ (사람 수, $1 \le M \le 1,000,000,000$)
- 각 심사대의 처리 시간 $T_i$

### **출력**

- `#{test_case} {최소 시간}`

---

## 💡 풀이 전략

1. **범위 설정**: $M$이 최대 10억이므로 결과값은 매우 큰 수가 될 수 있습니다. 초기 `high` 값을 넉넉하게 설정해야 합니다.
2. **조건 체크 함수**:
   ```python
   def can_process(time, target_people, counters):
       sum_people = 0
       for t in counters:
           sum_people += time // t
           if sum_people >= target_people: return True
       return sum_people >= target_people
   ```
3. **최종 답 도출**: `mid` 값이 조건을 만족할 때마다 `ans` 변수에 기록하고, 탐색이 끝난 후 최종 `ans`를 출력합니다.

---

## ⚠️ 주의 사항

- **자료형의 크기**: 결과값이 64비트 정수 범위를 넘을 수 있으므로 유의해야 합니다 (파이썬은 상관없음).
- **중단 조건**: `sum_people`을 더할 때 이미 $M$을 넘었다면 반복문을 조기 종료(break)하여 성능을 높일 수 있습니다.
