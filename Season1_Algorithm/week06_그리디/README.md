# 그리디(Greedy) 알고리즘이란?

- 탐욕스러운 방법 (탐욕 알고리즘)
- 미래를 생각하지 않고 지금 당장 최선인 것만 골라 나가는 전략

# 그리디 알고리즘의 핵심 개념

- 그리디 알고리즘은 전체적인 상황을 고려하기보다 **매 순간마다 가장 좋아 보이는 선택**을 반복하여 해답에 도달하는 방법
    - 특징: 계산 속도가 매우 빠르지만, 모든 문제에서 최적의 해를 보장하지는 않음
    - 성립 조건
        1. **탐욕적 선택 속성:** 현재의 선택이 나중의 선택에 영향을 주지 않아야 함
        2. **최적 부분 구조:** 부분 문제의 최적해들이 모여 전체 문제의 최적해가 됨

## 1. 정렬 기반 선택 (Sorting-based Selection)

많은 그리디 알고리즘의 첫 단추는 **데이터를 특정 기준에 따라 정렬**하는 것

- **핵심 아이디어:** 무작위 데이터를 가치가 높은 순, 혹은 비용이 낮은 순으로 정렬하면 매 순간의 '최선'을 찾기가 매우 쉬워짐
- **파이썬 팁:** **람다(lambda)**를 이용해 정렬 기준을 자유자재로 설정하는 것이 핵심

## 2. 활동 선택 문제 (Activity Selection Problem)

한 강의실에서 가장 많은 수업을 배정하거나, 한 회의실에서 가장 많은 회의를 열 수 있는 방법을 찾는 문제입니다.

- **탐욕적 선택:** **"종료 시간이 가장 빠른 활동"**부터 선택
- **이유:** 빨리 끝날수록 다음에 올 수 있는 활동의 선택 폭이 넓어지기 때문

## 3. 스케줄링 (Scheduling)

작업의 우선순위나 마감 기한이 있을 때 효율을 극대화하는 전략입니다.

- **대표 예시 (Minimum Latency):** 모든 작업의 대기 시간 합을 최소화하려면, 수행 시간이 짧은 작업부터 처리 (서비스 센터의 대기 줄을 줄이는 원리와 같음)
- **마감 기한이 있는 경우:** 각 단위 시간당 얻을 수 있는 이익이 가장 큰 작업을, 그 작업의 마감 기한에 가장 가까운 빈 시간에 배치하는 전략을 사용

## 4. 최소 비용 학습 (Minimum Spanning Tree & Prim/Kruskal)

여러 지점을 연결할 때 전체 비용을 최소로 만드는 경로를 찾는 것입니다. 보통 **크루스칼(Kruskal) 알고리즘**이 그리디의 정석으로 불립니다.

- **탐욕적 선택:** 모든 간선(Edge) 중 비용이 가장 적은 것부터 하나씩 선택
- **제약 조건:** 단, 선택했을 때 **사이클(Cycle)**이 생기면 안 됨 (이미 연결된 길을 또 연결할 필요는 없음)
