# SW 역량 및 학습 성취도 향상 스터디  :

## 1. 개요

**추진 배경 및 기획 의도
본 스터디는 초기 '알고리즘 풀이' 를 목적으로 시작되었으나, 참여자 전원의 희망 직무(PM, 교육, 데이터, 품질, AI)가 각기 다른 점을 고려할 때, 이후 계획했던 '기본 Computer Science 이론' 학습만으로는 개개인의 실무 역량 강화와 직접적인 연관성을 찾기 어렵다는 한계에 봉착했습니다.

이에 따라 획일화된 이론이나 암기식 학습을 넘어, 각자의 목표 직무에 특화된 SW 기술 스택과 도메인 지식을 깊이 있게 Deep-dive하고 서로 공유하는 실무 지향적 방식으로 스터디 방향을 전면 개편하였습니다

이러한 기획 의도를 바탕으로 SW 제품 생애주기 내의 핵심 5대 영역(기획, AI, 데이터, 품질, 교육) 심층 스터디를 통해 개별 직무 역량을 강화하고 SW 엔지니어링 성취도를 높이기 위한 정기 기술 공유 세션을 운영합니다.

* **목적:** SW 개발의 각 단계별 전문 스택(Stack)을 보유한 5인 체제의 기술 스터디
* **목표:**
  * **SW 기술 스택 고도화:** 각 직무별 필수 라이브러리, 프레임워크, 알고리즘 심층 학습
  * **학습 성취도 가시화:** 주 1회 기술 발표를 통해 개인별 학습 진척도를 점검하고 성과를 공유
  * **엔지니어링 협업 역량 강화:** 타 직무의 기술 프로세스를 이해하여 개발 생산성 및 소통 효율 극대화

## 2. 운영 계획

매주 **참여 인원(5명) 전원**이 본인의 주간 학습 테마와 SW 기술 연구 결과를 공유합니다.

* **시작 일시:** 2026년 3월 2주차 ~
* **진행 주기:** **매주 일 정기 운영**
* **발표 인원:** 전원 (5명)
* **소요 시간:** **약 50~60분** (인당 10분 내외)
* **형식:** 기술 중심 자유 주제 (라이브러리 분석, 아키텍처 연구, 트러블슈팅 사례 등)

## 3. SW 직무별 연구 주제 (상세 가이드)

각 직무의 전문성을 SW 엔지니어링 관점에서 접근하여 학습 성취도를 높입니다.

### 📋 PM (Software Product Management)

> **"SW 요구사항 명세 및 효율적인 아키텍처 설계 역량"**

* **요구사항 정의(PRD):** 유즈케이스(Use Case) 분석 기반의 서비스 명세 및 비즈니스 로직 설계
* **엔지니어링 협업:** API 명세서 읽기 및 작성법, 개발 생산성 향상을 위한 협업 툴(Figma-to-Code 등) 활용 연구
* **데이터 기반 기획:** SQL을 활용한 제품 지표 분석 쿼리 작성 및 데이터 모델링 기초 학습

### 🎓 교육 (Dev Education & Tech Writing)

> **"효율적인 지식 전달 및 클린 코드(Clean Code)를 위한 기술 가이드"**

* **커리큘럼 설계:** 특정 언어(React, Python, Node.js) 학습 로드맵 및 핵심 기술 노드(Node) 체계화
* **코드 리뷰 및 가이드:** 읽기 좋은 코드 작성을 위한 컨벤션 연구, 기술 문서(README, API Docs) 작성법 스터디
* **에듀테크 활용:** AI 페어 프로그래밍 도구 활용법 및 학습 데이터 분석을 통한 교육 성취도 측정 연구

### 📊 데이터 분석 (Data Engineering & Analytics)

> **"SW 성능 최적화를 위한 데이터 처리 및 분석 알고리즘 역량"**

* **데이터 엔지니어링:** 대용량 데이터 전처리를 위한 Pandas/Numpy 최적화 기법, 데이터 시각화 라이브러리 마스터
* **분석 알고리즘:** 통계적 가설 검정(A/B Test)의 수학적 원리 이해 및 Python 라이브러리 구현 실습
* **DB/쿼리 최적화:** 복잡한 SQL 쿼리 성능 개선 방안 및 인덱스(Index) 설계 원리 학습

### 🛠️ 품질 (Software & Data Quality Control)

> **"SW 결합 예측 모델링 및 테스트 자동화 기술 역량"**

* **품질 알고리즘:** 공정/SW 로그 데이터의 이상 탐지(Anomaly Detection)를 위한 머신러닝 모델링(LSTM, Isolation Forest 등)
* **데이터 관리 기술:** 제조 데이터 처리 시 발생하는 결측치/이상치 처리를 위한 통계적 기법 연구
* **품질 표준 연구:** ISO/IEC 등 글로벌 품질 표준을 준수하는 SW 데이터 설계 방법론 스터디

### 🤖 AI 엔지니어 & DevOps (MLOps & Infra)

> **"안정적인 SW 운영 인프라 구축 및 모델 서빙 자동화 역량"**

* **컨테이너 인프라:** Docker 기반의 개발 환경 구축 및 Kubernetes 오케스트레이션 기초 개념 연구
* **MLOps/CI-CD:** GitHub Actions를 활용한 자동화 파이프라인 구축, AI 모델의 FastAPI 서빙 최적화 실습
* **최신 AI 아키텍처:** LLM(Llama, GPT 등) 모델의 구조 분석 및 임베딩(Embedding) 벡터 DB 연동 기술 스터디

## 4. 기대 효과

1. **학습 성취도 극대화:** 타인에게 설명하는 과정을 통해 학습한 SW 개념을 완벽히 내재화
2. **풀스택적 시야 확보:** 기획부터 배포까지의 SW 전체 사이클을 이해하는 엔지니어로 성장
3. **가시적 성과 도출:** 매주 생성되는 기술 블로그 형태의 요약본을 포트폴리오로 자산화
4. **발표 및 커뮤니케이션 역량 강화:** 정기적인 발표를 통해 자신의 생각을 논리적으로 표현하고 전달하는 스피치 역량 향상

## 6. 자료 아카이빙 규칙

스터디의 모든 산출물은 GitHub 레포지토리에 투명하게 아카이빙하여 공동의 포트폴리오로 활용합니다.

### 📁 폴더 및 파일명 컨벤션

발표 자료(PDF, PPT, Markdown 등)와 실습 코드 등은 **'주차별 폴더'** 단위로 모아 관리합니다.

* **폴더명 규칙:** `[주차]_YYYYMMDD` (예: `1주차_20260315`)
* **발표 자료 파일명:** `[직무명]_[이름]_발표주제.확장자`
  * 예시: `PM_홍길동_요구사항명세가이드.pdf`
  * 예시: `데이터_김데이터_Pandas최적화.md`

### 💬 Commit 메시지 형식

규칙적인 커밋 메시지로 히스토리를 깔끔하게 관리합니다.

* **기본 형식:** `docs: [직무명] N주차 - 핵심 주제 키워드`
* **커밋 예시:**
  * `docs: [PM] 1주차 - API 명세서 작성법 문서 추가`
  * `feat: [AI] 2주차 - FastAPI 기반 모델 서빙 실습 코드 업로드`
  * `chore: [공통] 3주차 README 일정 최신화`
