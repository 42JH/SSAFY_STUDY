### 1. 데이터 분류

#### 1) 구조화(저장 형태)에 따른 분류

| 구분 | 핵심 특징 | 예시 |
| --- | --- | --- |
| 정형(Structured) | 미리 정해진 **스키마**에 맞춰 저장. 구조와 내용이 분리되어 관리 | 엑셀 스프레드시트, RDB 테이블 |
| 반정형(Semi-structured) | 구조는 있으나 **메타데이터가 데이터 안에 포함**. 구조 파악을 위한 **파싱** 필요 | HTML, XML, JSON, 센서 데이터 |
| 비정형(Unstructured) | 정해진 구조 없이 저장 | 소셜 텍스트, 영상, 이미지, 워드/PDF |

#### 2) 값의 특성에 따른 분류

- 범주형 데이터 (Categorical / Qualitative)
    - 범주로 구분되는 값(종류)을 가지는 데이터
    - 크기 비교 및 산술 연산이 불가
    - 주관적 생각, 평가가 포함될 수 있음
    - 종류
        - **명목형(Nominal)**: 서열(순서) 없음 (예: 성별, 혈액형, 학과, 거주지역, 음식 메뉴, MBTI)
        - **순서형(Ordinal)**: 서열(순서) 있음 (예: 학년, 학점, 회원 등급)
- 수치형 데이터 (Numerical / Quantitative)
    - 숫자 값으로 표현되며, 크기 비교 및 산술 연산 가능
    - 객관적 측정을 통해 표현되는 경우가 많음
    - 종류
        - **이산형(Discrete)**: 개수를 셀 수 있는 불연속 값 (예: 고객 수, 판매량, 합격자 수)
        - **연속형(Continuous)**: 측정으로 얻는 연속 값 (예: 키, 몸무게, 온도, 점수)

### 3. 정보 시스템과 데이터베이스

#### 정보 시스템(Information system)

- 조직 운영에 필요한 데이터를 수집·저장하고, 필요 시 유용한 정보를 만들어 주는 수단

#### 데이터베이스(DB; Database)

- 정보 시스템 내에서 데이터를 저장하고, 필요할 때 제공하는 역할

#### 데이터베이스(DB)의 정의

- 특정 조직의 여러 사용자가 **공유**하여 사용할 수 있도록 **통합**해서 저장한 운영 데이터의 집합
- 구성 요소
    - **공유 데이터(Shared data)**: 조직의 여러 사용자가 함께 소유·이용 가능한 공용 데이터
    - **통합 데이터(Integrated data)**: 최소 중복 + 통제 가능한 중복만 허용
    - **저장 데이터(Stored data)**: 컴퓨터가 접근 가능한 매체에 저장된 데이터
    - **운영 데이터(Operational data)**: 조직의 주요 기능 수행에 지속적으로 필요한 데이터

---

### 4. 데이터베이스(DB)의 특징

1. **실시간 접근(Real-time accessibility)**
    - 사용자 요구에 실시간으로 응답
2. **계속 변화(Continuous evolution)**
    - 삽입/삭제/수정을 통해 최신의 정확한 데이터 유지
3. **동시 공유(Concurrent sharing)**
    - 서로 다른 데이터의 동시 사용뿐 아니라 같은 데이터의 동시 사용도 지원
4. **내용 기반 참조(Content reference)**
    - 주소/위치가 아니라 **내용**으로 참조 가능
    - 예: 재고량이 1,000개 이상인 제품의 이름을 검색하시오.

### 5. 파일 시스템 vs DBMS

#### 파일 시스템의 문제점

- **데이터 종속성(Data dependency)**
    - 응용 프로그램이 데이터 파일에 종속적
    - 파일의 구조/구성/접근 방법 변경 시, 응용 프로그램도 함께 변경 필요
- 데이터 파일에 대한 동시 공유, 보안, 회복 기능이 부족
- 응용 프로그램 개발 및 유지보수가 어려움

<aside>
✅

**해결 방향**: 데이터 통합 (파일 시스템 → 데이터베이스 시스템)

</aside>

#### DBMS(DataBase Management System)

- 정의: 기존 파일 시스템의 문제를 해결하기 위해 제시된 소프트웨어. 조직에 필요한 데이터를 DB에 **통합·저장·관리**
- 예: Oracle, MySQL, MS-SQL, MariaDB

#### DBMS의 기능

- **데이터 정의(DDL)**: 데이터베이스 구조를 정의 또는 수정
- **데이터 조작(DML)**: 데이터 삽입·수정·검색
- **데이터 제어(DCL)**: 정확하고 안전하게 유지(권한/무결성/동시성 등)

#### DBMS의 장점과 단점

**장점**

- 데이터 중복 통제
- 데이터 독립성 확보
- 데이터 동시 공유
- 데이터 보안 향상
- 데이터 무결성 유지
- 표준화된 방식으로 접근
- 장애 회복 지원
- 개발 및 유지 비용 감소

**단점**

- 고비용(구매/사용자 수에 따라 증가)
- 백업과 회복 방법이 복잡
- 중앙 집중 관리로 인한 취약점(장애 시 전체 업무 중단)
