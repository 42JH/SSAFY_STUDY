# 스터디 9주차 발표 (교육 파트)

## 주제 : 지식 추적(Knowledge Tracing)을 활용한 개인화 학습 경로 설계

### 0. 지난 발표에서는…

- **7주차 발표 (Clean Code & Technical Writing):** **'코드를 어떻게 올바르게 작성할 것인가'**라는 쓰기(Writing)의 관점에서 접근했음.
- **8주차 발표 (AST & 정적 분석):** 작성된 코드를 기계가 어떻게 **구조적으로 '읽고 분석(Analysis)'할 것인가**를 다루었음.
- **이번 9주차 발표:** 앞서 분석한 데이터(AST, 정적 분석 결과)를 바탕으로, **'학습자의 성장을 어떻게 이끌어내고 개인화된 교육 경로를 설계할 것인가'** 에 대해 알아보도록 하자.

### 1. 문제 제기

**획일화된 교육의 한계**

- 모든 학생은 취약한 지점(Concept Node)이 다름
- 동일한 문제를 풀어도 누구는 '문법'이 부족하고, 누구는 '알고리즘'이 부족함
- 강사가 수십 명의 학습 상태를 실시간으로 파악하여 맞춤형 과제를 내주는 것은 불가능

**👉 핵심 문제**
“학습자의 과거 코드 제출 이력을 바탕으로, 다음에 풀어야 할 최적의 문제는 무엇인지 예측할 수 있는가?”

### 2. 해결 아이디어

**학습 데이터를 기반으로 한 '지식 상태'의 수치화**

- 학습자가 특정 기술(예: 반복문, 재귀함수)을 마스터했는지 확률적으로 계산
- 단순 정답 여부를 넘어, AST 분석을 통해 드러난 '나쁜 습관'까지 데이터화

**👉 질문**
“컴퓨터는 어떻게 학습자의 머릿속에 있는 지식의 양을 측정할까?”

### 3. 핵심 개념 ①

**지식 추적 (Knowledge Tracing)**

- 학습자의 과거 학습 이력을 바탕으로 특정 지식 구성 요소(Knowledge Component)에 대한 숙련도를 모델링하는 기술

**핵심 원리: BKT (Bayesian Knowledge Tracing) :** 
학습자의 문제 풀이 이력을 기반으로, 특정 개념에 대한 숙련도를 확률적으로 추정하고 업데이트 하는 모델

BKT 모델은 학습자의 상태를 4가지 파라미터로 분석한다.

| **변수명** | **의미** | **설명** |
| --- | --- | --- |
| **P(L0)** | **Initial Knowledge** | 학습 전 이미 해당 개념을 알고 있을 확률 |
| **P(T)** | **Transition** | 문제를 풀면서 지식을 새로 습득할 확률 |
| P(G) | **Guess** | 모르지만 우연히 맞출 확률 (찍기) |
| P(S) | **Slip** | 알지만 실수로 틀릴 확률 |

**핵심 메시지**
• 한 번 맞췄다고 '안다'고 단정하지 않음
• 반복적인 관찰(문제 풀이 데이터)을 통해 숙련도 확률을 업데이트함

### 4. 핵심 개념 ② : 지식 그래프와 선수 학습

- **정의**: 지식 간의 선후 관계를 노드(Node)와 연결선(Edge)으로 시각화한 지도
- **교육적 활용**:
    - 학습자의 오류를 단순히 결과가 아닌 **근본 원인(선수 개념 부족)**으로 추적 가능
    - 특정 개념에서 반복적으로 실패할 경우, 해당 개념이 아닌 **선행 개념으로 학습 경로를 자동 조정**
    - 각 학습자의 지식 상태에 따라 **서로 다른 학습 경로를 동적으로 생성** (Adaptive Learning)
    - 불필요한 반복 학습을 줄이고, **취약한 개념에 집중하는 효율적인 학습 가능**
    - 예시 :
    '정렬 알고리즘' 문제를 반복해서 틀리는 학생의 AST 분석 결과,
    '배열 인덱스 접근' 오류가 잦다면 →
    상위 개념인 정렬이 아닌 하위 개념인 **'배열'** 단계로 학습 경로를 즉시 수정

### 5. 개념의 연결: AST 분석 + 지식 추적

**데이터 기반의 개인화 교육**

1. **코드 분석:** 8주차에 배운 AST 분석을 통해 학생 코드에서 사용된 문법과 오류 패턴 추출
2. **지식 업데이트:** 추출된 데이터를 지식 추적 모델에 입력하여 '반복문', '조건문' 등의 숙련도 점수 갱신
3. **학습 경로 설계:** 현재 점수를 기반으로 '가장 적절한 난이도'의 다음 과제나 이론 자료를 추천

### 6. 예시: 맞춤형 피드백과 경로 추천

- **사례 A (기초 부재):** AST 분석 결과 문법 오류가 잦음 → 지식 추적 모델이 '기초 문법' 숙련도를 낮게 측정 → **문법 퀴즈 우선 배정**
- **사례 B (효율성 부족):** 문법은 맞으나 중첩 루프를 남용함 → '시간 복잡도' 노드 숙련도 정체 감지 → **알고리즘 최적화 가이드 제시**
- **사례 C (개념은 이해했지만 응용 부족)**: 기본 문법과 단일 개념 문제는 잘 해결하지만, 여러 개념이 결합된 문제에서는 오답이 증가 → 지식 추적 모델이 ‘개념 간 연결(응용력)’ 숙련도가 낮다고 판단 → **단일 개념 문제가 아닌, 단계적으로 난이도가 상승하는 종합 문제 추천**

### 7. 교육적 가치

1. **완전 학습 구현:** 부족한 부분을 확실히 채워야 다음 단계로 넘어가는 체계적 관리
2. **객관적 지표:** 강사의 주관이 아닌, 누적된 데이터를 바탕으로 한 일관된 평가
3. **학습 효율 극대화:** 아는 부분은 빠르게 넘어가고 모르는 부분에 집중하여 학습 시간 단축
