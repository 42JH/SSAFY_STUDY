# 스터디 10주차 발표 (교육 파트)

**주제 : LLM 기반의 지능형 튜터링 시스템(ITS)과 자동 피드백 생성 기술**

### 0. 서론: 이전 발표에서는…

- **7주차 Clean Code & Technical Writing:** '코드를 어떻게 올바르게 작성할 것인가'라는 인간의 **작성(Writing)** 관점.
- **8주차 AST & 정적 분석:** 작성된 코드를 구조적으로 **분석(Analysis)**하여 논리적 오류를 잡아내는 기계의 관점.
- **9주차 Knowledge Tracing:** 분석된 데이터를 축적하여 학습자의 **지식 상태(State)**를 확률적으로 모델링하는 관점.
- **이번 10주차 발표:** 앞선 단계들에서 파악된 '학습자의 취약점'을 해결하기 위해, **LLM(대형 언어 모델)**을 활용하여 인간 튜터처럼 정교한 **맞춤형 피드백을 실시간으로 생성**하는 방법을 다룰 것이다.

### 1. 문제 제기: "정답은 알겠는데, 왜 틀렸는지는 모르겠어요"

- **기본적인 채점 시스템의 한계:** 현재 대부분의 온라인 코딩 플랫폼은 `Pass/Fail` 혹은 `Wrong Answer`라는 결과만 제공한다.
- **튜터의 부재:** 학습자는 본인의 코드가 왜 비효율적인지, 어떤 개념이 꼬였는지 파악하기 위해 구글링이나 커뮤니티에 의존해야 하며, 이 과정에서 학습 흐름이 끊김.
- **확장성 문제:** 1명의 교육자가 수백 명의 코드 스타일과 논리 구조를 일일이 분석하여 개별 피드백을 주는 것은 물리적으로 불가능함.
- 👉 **핵심 질문:** "AST로 분석된 구조적 결함과 지식 추적 모델로 파악된 취약점을 바탕으로, **학습자의 수준에 딱 맞는 설명**을 자동으로 만들어낼 수 있는가?"

### 2. 핵심 개념: 지능형 튜터링 시스템 (ITS, Intelligent Tutoring Systems)

- **정의:** 컴퓨터 기술을 활용하여 학습자에게 1:1 개인 교습과 유사한 교육 환경을 제공하는 시스템
- **구성 요소:** 
1. **도메인 모델:** 가르칠 지식(CS 지식 그래프).
2. **학생 모델:** 학습자의 현재 수준(9주차의 지식 추적 결과).
3. **교수 모델:** 어떤 타이밍에 어떤 힌트를 줄 것인가에 대한 전략.
4. **인터페이스 모델:** 학습자와의 상호작용 도구(LLM 기반 챗봇 등).

### 3. 핵심 기술: LLM을 이용한 피드백 생성 (Socratic Questioning)

- **단순 해설 vs 소크라테스식 질문:** LLM은 단순히 정답 코드를 짜주는 것이 아니라(단순 해설), 학습자가 스스로 생각하게 만드는 '힌트'를 생성하는 데 최적화되어 있음(소크라테스식 질문).
- **Chain-of-Thought (CoT) Prompting:** 모델이 피드백을 생성할 때, 학습자의 AST 분석 결과와 과거 오답 이력을 참고하여 논리적인 추론 과정을 거치도록 유도한다.
- **예시:**
    - *잘못된 피드백:* "8번 라인에서 `for`문 범위를 `n-1`로 고치세요." (정답 노출)
    - *지능형 피드백:* "현재 작성하신 반복문이 배열의 마지막 요소까지 방문하고 있나요? 인덱스 범위를 다시 한번 확인해보면 좋겠어요." (사고 유도)

### 4. 활용 방안: AST + 지식 추적 + LLM의 결합

- **1단계 (입력):** 학습자가 코드를 제출
- **2단계 (분석):** **AST**를 통해 코드의 구조적 특징(중첩 루프 사용 여부, 변수 명명 등)을 추출
- **3단계 (진단):** **지식 추적 모델**이 해당 학생이 현재 '시간 복잡도' 개념이 약하다는 것을 인지했다고 가정
- **4단계 (생성):** **LLM**이 위 데이터를 종합하여, "당신은 현재 시간 복잡도 최적화에 어려움을 겪고 있군요. 이 코드의 중첩 루프를 줄일 수 있는 다른 자료구조가 있을까요?"라는 맞춤형 힌트를 생성

### 5. 활용 예시: 에듀테크 플랫폼의 'AI 코칭 봇'

- **상황:** '재귀 함수' 문제를 푸는 학생이 기저 사례(Base Case)를 빠뜨려 무한 루프에 빠진 경우.
- **시스템 동작:**
    1. **AST 분석:** 함수 내부에서 자기 자신을 호출하나, 조건문(if) 구조가 미비함을 파악.
    2. **지식 추적:** 이 학생은 이전에도 '조건문' 노드에서 낮은 점수를 받았음을 확인.
    3. **LLM 피드백:** "재귀 함수는 언제 멈춰야 할까요? 이전 학습에서 다룬 '조건문'을 활용해 멈춤 신호를 만들어보세요!"라는 힌트와 관련 시각화 자료를 제공.

### 6. 의의 및 한계

- **의의:**
    - **24/7 가용성:** 시간과 장소에 구애받지 않는 개인 과외 선생님 역할.
    - **정서적 지지:** 틀렸을 때 비난하지 않고 꾸준히 격려하는 피드백 제공 가능.
    - **데이터 기반 성장:** 학습자의 성장을 수치화하고 그에 걸맞은 난이도를 조절하여 '몰입(Flow)' 상태 유지.
- **한계:**
    - **환각 현상(Hallucination):** LLM이 문법적으로는 완벽하지만 논리적으로 틀린 피드백을 줄 위험.
    - **의존성 문제:** 학습자가 스스로 고민하기보다 AI 힌트에만 의존하게 될 가능성.

### 7. 결론

결국 교육 기술의 진화는 **'데이터를 어떻게 교육적 가치로 전환할 것인가'**의 문제이다.
**좋은 코드**(7주차)를 기준으로 삼아, **기계적으로 분석**(8주차)하고, **학습 상태를 추적**(9주차)하여, 최종적으로 **지능형 피드백**(10주차)을 주는 하나의 유기적인 시스템이 완성될 때, 비로소 진정한 의미의 '개인화 교육'이 가능해진다.
